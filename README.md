# trip-crawler

## Workflow

1. В Projects берёте задачу на себя и переносите в In Progress
1. Конвертируете задачу в issue в Projects
1. Создаёте ветку в формате `feature/<platform>/<feature-name>`, где `platform` - платформа задачи (backend, frontend, devops), `feature-name` - короткое название фичи в kebab-case
1. Коммиты называете в формате `Closes #<номер issue из задачи>: <описание того, что сделал в коммите>`
1. Создаёте PR в master, ждёте ревью, правитие замечания, мержите в мастер
