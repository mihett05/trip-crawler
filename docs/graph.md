# Модель данных для графовой БД

## Концептуальная схема

### Вершины
- City: Населенный пункт. Точка входа для поиска (например, "Пермь")
- Station: Конкретный транспортный узел (Аэропорт Большое Савино, Вокзал Пермь-2)
- Trip: Конкретный экземпляр перемещения (Рейс SU-100, Поезд 001А). Содержит время, цену и тип. Я не уверен, что свойства на ребрах индексируются, поэтому вынесем перемещения в вершины

### Ребра
- City.has_station $\rightarrow$ Ссылка на вокзалы/аэропорты города
- Station.departs $\rightarrow$ Ссылка на Trip (исходящий рейс)
- Trip.destination $\rightarrow$ Ссылка на Station (куда прибывает рейс)
